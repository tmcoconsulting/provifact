When this static build is served by the reviewed Worker, the panel calls same-origin
`/api/status` and `/api/narrative`. Fixture mode performs no OpenAI request. OpenAI mode is visibly
labeled before a user initiates generation and never accepts a browser-provided key.

returned capacity unavailable during the bounded live validation. The service key remains
server-side; EvidenceOps does not retry or silently substitute fixture output after a live failure.

## End-to-end walkthrough

| Stage | Demonstrated result | Boundary |
| --- | --- | --- |
| Reviewed desired state | Four small vendor-neutral macOS records at `synthetic-reviewed-commit` | Git intent |
| Observation | Three normalized, synthetic settings | Provider-neutral evidence |
| Drift | Match, difference, missing evidence, and unsupported evaluator | Deterministic algorithm |
| Private package | Created in memory only for this demo | Never written to static assets |
| Publication | Policy `evidenceops-publication-v1.0.0` passed twice | Public boundary |
| Narrative | Offline fixture follows the GPT-5.6 structured contract | AI-shaped, not a model call |
| Verification | Typed status claims verified; all generated prose quarantined | Deterministic verifier |

## See both verifier boundaries

For the normal fixture, the verifier records one typed `finding_status` claim for each exact
finding ID. It does not mark executive summaries, explanations, impact, limitations, or review
questions as verified. Those fields are listed as generated prose quarantined for human review.

The adversarial fixture changes only the executive summary to claim the environment is compliant.
The verifier returns `accepted: false` with reason `unsupported compliance or certification
verdict`. See the [rejected result](assets/data/phase1-rejected-verification.json).

This makes the design choice visible: GPT may explain evidence, but it is not trusted to establish
the evidence or decide compliance.

## Reproduce it

```bash
python -m evidenceops run-demo --output-dir build/synthetic-demo
python scripts/check_public_artifacts.py build/synthetic-demo
```

There is no automatic remediation. A Git revert changes desired-state history only; it does not
revert Intune. Human assessors retain final judgment.
